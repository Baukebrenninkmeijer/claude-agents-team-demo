Three Agent Orchestration Patterns

How coordination flows between agents — and who pays for it.

Single Agent

One loop · one context

Sequential tool calls. One conversation, one window. Default choice — fits more work than people think.

Subagents

Hub-and-spoke · cache-friendly

Fork, work, report back. Byte-identical context means all forks hit cache. Parallelism is nearly free — but subs can't talk to each other.

Team of Agents

Peer-to-peer · full context each

Direct messaging. Independent context windows, shared task list. Peers challenge reasoning and reach consensus — at 3–15x the cost.
